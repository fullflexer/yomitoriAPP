このスキルは、相続関係説明図アプリの要件定義を **Claude Code 向けの作法** でまとめるためのものです。

## 実行前に必ず行うこと
1. `CLAUDE.md` を読む
2. `prompts/souzoku-requirements-2026.md` を読む
3. `templates/doc-set-outline.md` を読み、期待する成果物の粒度を把握する
4. 既存の `docs/` や入力資料があれば先に確認する

## 進め方
1. まず現状理解
   - 既存ファイル
   - 既存要件
   - 入力資料
   - 未解決論点
   を短く要約する

2. 次に前提整理
   - 情報不足があっても停止しない
   - 重要確認事項のみ最大3件
   - それ以外は `仮定:` として明示して進む

3. その後、以下の成果物を作成または更新する
   - `docs/requirements-definition.md`
   - `docs/edge-cases.md`
   - `docs/tech-risk-register.md`
   - `docs/roadmap.md`
   - `docs/open-questions.md`

4. 書き方
   - 日本語
   - 実務者がレビューしやすい構造
   - 各要件は可能な限り Must / Should / Could を付与
   - 未確認の法令・実務運用は断定しない
   - OCR / LLM / ルールベース / 人手補正の責務分離を明示する

5. 最後に短くまとめる
   - 何を作成 / 更新したか
   - 主要な仮定
   - 残る高リスク
   - ステークホルダー確認事項

## 特に重視する観点
- 戸籍の高機微情報としての取り扱い
- 名寄せと時系列整合性
- 相続判定ロジックの説明可能性
- 実務運用での補正フロー
- 2026年時点での導入現実性
- 士業事務所での運用保守性
