# CLAUDE.md

## このリポジトリの目的
日本の相続実務向けに、戸籍謄本・除籍謄本・改製原戸籍等を OCR / 画像解析で読み取り、人物・身分事項・続柄を構造化し、法定相続人判定と「相続関係説明図」の生成を支援する Web アプリの要件定義を行う。

## このリポジトリで Claude Code に期待する役割
- まず既存資料・既存ファイルを読み、前提・不足情報・制約を整理する
- いきなり実装に飛ばず、先に `docs/` 以下の要件成果物を整備する
- 不足情報があっても作業停止せず、合理的な仮定を明示して前進する
- 法務・戸籍実務・登記実務に関わる論点は、**断定よりも根拠範囲の明示** を優先する
- 正確な条文番号・運用基準が未確認なら、推測で埋めず `要確認` と明示する

## 出力ポリシー
- 主要成果物は Markdown で作成する
- 文体は日本語、簡潔だが省略しすぎない
- 各要件はできるだけ次の情報を持つ
  - 要件ID
  - 優先度（Must / Should / Could）
  - 要件本文
  - 背景 / 根拠
  - 受入基準
  - 依存関係または前提
- 曖昧な箇所は `仮定:`、未決事項は `未決:`、法務確認事項は `法務確認:` と明示する

## 作業順序
1. 既存ファイルと入力資料を確認
2. スコープ・前提・対象ユーザー・対象書類を整理
3. 要件定義書を 8 カテゴリで作成
4. エッジケース一覧・技術リスク評価表・フェーズ計画・未決事項リストを分離して作成
5. 技術選定は **2026年時点で実務運用可能** かつ **日本国内の相続実務に耐える** 観点で比較する
6. 実装案は、監査性・説明可能性・人間補正フロー・個人情報保護を優先する

## 禁止事項
- 未確認の条文番号や実務運用を断定しない
- OCR / LLM の精度を保証しない
- 例外事例を無視して単純化しすぎない
- 法務判断が必要な事項を、実装仕様だけで確定扱いしない

## 技術検討の軸
- OCR 精度
- 手書き・縦書き・旧字体・改製原戸籍への耐性
- 名寄せと時系列整合性
- 相続判定ロジックの検証容易性
- 監査ログと再現性
- 高機微個人情報の保護
- 士業事務所での導入 / 運用しやすさ
- ベンダーロックインと運用コスト
