# yomitoriAPP エッジケース一覧

本書は AC-2 対応のエッジケース整理表であり、[requirements-definition.md](/Users/chion/Projects/yomitoriapp/docs/requirements-definition.md) の `REQ-201`、`REQ-202`、`REQ-203`、`REQ-204`、`REQ-304` を主参照とする。

- 難しさは `最高` / `高` / `中` で評価する。
- MVP 区分は [requirements-definition.md](/Users/chion/Projects/yomitoriapp/docs/requirements-definition.md) の明示的な MVP スコープ記載を優先する。
- 法令未確認事項や運用差があり得る事項は `要確認` と明示する。
- レビュー要否は、法務・士業レビューの必要度を `必須` / `推奨` / `不要` で示す。

| ケース名 | 難しさ | 判定上の注意点 | 実装上の注意点 | レビュー要否 |
| --- | --- | --- | --- | --- |
| 代襲相続（直系卑属・無限） | 最高（MVP内） | 根拠条文: 民法887条2項・3項。子が相続開始前に死亡した場合、又は欠格・廃除で相続権を失った場合に、その直系卑属が代襲する。再代襲は直系卑属方向に続く整理を前提とするが、戸籍記載と事実関係の対応は要確認。 | 被代襲者ごとに `死亡 / 欠格 / 廃除 / 放棄` を別コードで保持し、代襲原因として使うのは `死亡 / 欠格 / 廃除` のみと明示する。`放棄` は別ケースとして除外判定へ送る。`要件参照: REQ-202, REQ-203, REQ-204, REQ-304`。 | 必須 |
| 代襲相続（傍系・甥姪止まり） | 高（MVP内） | 根拠条文: 民法889条2項、887条2項準用。兄弟姉妹が被相続人より先に死亡し、又は欠格・廃除で相続権を失った場合は甥姪が代襲するが、再代襲はしない整理を前提とする。甥姪以遠は対象外。 | 傍系枝は再帰深度を 1 代に制限し、甥姪までで打ち切る。直系代襲ロジックの流用時に無限再帰させない。`要件参照: REQ-202, REQ-204`。 | 必須 |
| 普通養子（実親の相続権維持） | 高（MVP外） | 根拠条文: 民法809条。普通養子は養親の嫡出子の身分を取得する一方、実親側との相続関係が残る整理が一般的だが、個別の身分関係・代襲範囲は要確認。 | 普通養子と特別養子を別属性で保持し、普通養子で実親側判定が絡む場合は自動確定しない。`要件参照: REQ-202, REQ-203, REQ-304`。 | 必須 |
| 特別養子（実親関係消滅） | 高（MVP外） | 根拠条文: 民法817条の9。特別養子は実方の父母及びその血族との親族関係が終了する。ただし、同条ただし書の例外運用は要確認。 | 養子種別、成立日、例外有無を明示的に保持する。不明な戸籍表現は通常養子扱いに寄せず、停止してレビューに回す。`要件参照: REQ-202, REQ-203, REQ-304`。 | 必須 |
| 半血兄弟姉妹（相続分1/2） | 高（MVP内） | 根拠条文: 民法900条4号。父母の一方のみを同じくする兄弟姉妹の相続分は、全血兄弟姉妹の 2 分の 1 とする。順位判定と相続分計算を混同しない。 | 兄弟姉妹の順位判定後に、相続分計算で重み `全血=1 / 半血=0.5` を適用する。`要件参照: REQ-202, REQ-204`。 | 推奨 |
| 数次相続 | 最高（MVP外） | 根拠条文: 民法882条、896条を起点に整理するが、数次相続自体の運用整理は要確認。被相続人死亡後に相続人が死亡すると、二つの相続を時系列で分けて扱う必要がある。 | 1 件として潰さず、一次相続と二次相続を別イベント列・別計算単位で保持する。MVP では自動確定対象外とする。`要件参照: REQ-203, REQ-401, REQ-403`。 | 必須 |
| 胎児の相続権 | 高（MVP内） | 根拠条文: 民法886条。胎児は相続について既に生まれたものとみなされるが、死体で生まれたときは適用しない。懐胎時点の認定資料は要確認。 | 出生確定前は「仮相続人」状態で保持し、死産・生産の確定で再計算する。未確定のまま帳票に確定表示しない。`要件参照: REQ-202, REQ-304, REQ-401`。 | 必須 |
| 失踪宣告 | 高（MVP内） | 根拠条文: 民法30条、31条。普通失踪は 7 年、生死不明の危難失踪は危難去後 1 年を経て宣告対象となり、死亡みなし時点は宣告日ではなく法定時点である。取消しの扱いは民法32条も要確認。 | 宣告日と死亡みなし日を別管理し、相続開始時点の計算には死亡みなし日を使う。審判書未確認の案件は停止する。`要件参照: REQ-202, REQ-203, REQ-304`。 | 必須 |
| 相続放棄と代襲の関係 | 最高（MVP外） | 根拠条文: 民法939条、887条2項、889条2項。相続放棄をした者は初めから相続人とならなかったものとみなされるため、欠格・廃除と異なり代襲原因にならない整理を前提とする。 | `放棄 / 欠格 / 廃除 / 死亡` を同じ「権利喪失」で束ねない。放棄受理証明書等の外部証憑がない限り自動確定しない。`要件参照: REQ-203, REQ-204, REQ-304`。 | 必須 |
| 相続欠格・推定相続人廃除 | 最高（MVP外） | 根拠条文: 民法891条、892条。欠格は法定事由該当で相続権を失い、廃除は家庭裁判所手続を伴う。代襲可否や効力発生時点の個別事情は要確認。 | 欠格と廃除は証憑要件もワークフローも別にする。証憑未確認なら判定停止にし、理由を説明文へ出す。`要件参照: REQ-203, REQ-304, REQ-401`。 | 必須 |
| 認知（死後認知含む） | 高（MVP外） | 根拠条文: 民法787条。子、その直系卑属等は認知の訴えを提起でき、父又は母の死亡後 3 年以内の死後認知が問題となり得る。認知の効力が相続人範囲へ及ぶ時点整理は要確認。 | 認知の有無だけでなく、認知成立日、判決・和解・戸籍反映日を別管理する。死後認知が絡む案件は自動確定しない。`要件参照: REQ-203, REQ-304, REQ-403`。 | 必須 |
| 同時死亡推定 | 高（MVP内） | 根拠条文: 民法32条の2。死亡の先後が明らかでない者同士は同時死亡と推定され、相互に相続しない整理が基本となる。代襲との接続は個別事案で要確認。 | 死亡イベントに「時刻確定 / 推定 / 不明」の確度を持たせ、先後不明なら同時死亡ルールへ分岐する。代替シナリオを勝手に確定しない。`要件参照: REQ-202, REQ-204, REQ-304`。 | 必須 |
| 再婚と連れ子 | 中（MVP内） | 根拠条文: 民法725条、727条、887条1項。婚姻だけでは連れ子は姻族にとどまり、養子縁組がなければ被相続人の「子」として扱わない整理を前提とする。 | 「配偶者の子」と「法的な子」を別概念で保持し、戸籍の養子縁組記載がない限り相続人候補へ自動昇格させない。`要件参照: REQ-201, REQ-202, REQ-304`。 | 推奨 |
| 離婚後の子の相続権 | 中（MVP内） | 根拠条文: 民法887条1項。離婚により配偶者性は失われても、子が被相続人の「子」であること自体は当然には消えない整理を前提とする。親権・監護と相続権の切り分けは要確認。 | 離婚イベントで消すのは配偶者関係のみとし、親子リンクを自動削除しない。親権者変更情報を相続人判定の除外条件に使わない。`要件参照: REQ-201, REQ-401, REQ-402`。 | 推奨 |
| 外国籍相続人 | 最高（MVP外） | 根拠条文: 法の適用に関する通則法36条。相続は被相続人の本国法によるため、相続人の外国籍そのものより被相続人の国籍・本国法確認が重要である。反致、複数国籍、無国籍、常居所の論点は要確認。 | 国籍情報だけで日本民法ロジックへ流し込まない。外国法調査が必要な案件として停止し、必要資料とレビュー要件を明示する。`要件参照: REQ-203, REQ-304`。 | 必須 |
