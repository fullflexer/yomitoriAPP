# yomitoriAPP 未解決論点一覧

本書は `docs/requirements-definition.md` の `OQ-1`〜`OQ-5` を実体化した未解決論点一覧である。法令・実務運用・受理基準の未確認事項は断定せず、`要確認` と明示する。

要求仕様で指定された `REQ-016`〜`REQ-019`、`REQ-027`〜`REQ-030` などの旧採番は、現行の `docs/requirements-definition.md` には存在しない。本書では旧採番を残しつつ、現行要件IDへの対応関係を併記し、差分は `要確認` とする。

| OQ-ID | 論点 | 理由（なぜ未決か） | 決定者 | 期限 | 影響範囲 |
| --- | --- | --- | --- | --- | --- |
| <a id="oq-1"></a>OQ-1 | 外部API利用時の個人情報保護法上の整理（委託 vs 第三者提供） | 要確認。Claude Vision 等の外部APIに戸籍データを送信する行為が個人情報保護法上「委託」か「第三者提供」かで必要な対応が異なる。委託なら DPA 締結や監督義務が中心だが、第三者提供なら本人同意や外国にある第三者提供の整理が必要になる。 | 法務弁護士 + プロダクトオーナー | Phase 2 Go/No-Go 判定前（`REQ-602`, `REQ-702`, `RISK-004`, `RISK-010` の確定前。日付は要確認） | `REQ-602`, `REQ-504`, `RISK-004`, `RISK-010`, プライバシーポリシー全体, `docs/roadmap.md` Phase 2〜4（要確認: 指定仕様の `REQ-028`〜`REQ-030` との対応付け） |
| <a id="oq-2"></a>OQ-2 | 相続判定誤りの免責・責任分界 | 要確認。AIが誤った相続人判定を出力し、士業が依拠して法務局に提出した場合の責任の所在が未確定である。過度な免責は士業ユーザーの採用を阻害しうるため、契約・UI・出力物での責任分界の水準を決め切れていない。 | 法務弁護士 + プロダクトオーナー | Phase 3 着手前（`REQ-603`, `REQ-703`, `RISK-007` の運用反映前。日付は要確認） | `REQ-203`, `REQ-603`, `RISK-007`, 利用規約, UI上の免責表示, `docs/roadmap.md` Phase 3〜4（要確認: 指定仕様の `REQ-029` との対応付け） |
| <a id="oq-3"></a>OQ-3 | 戸籍データの保持期間・削除ポリシー | 要確認。案件完了後にどの期間データを保持するかは、個人情報保護法上の安全管理措置と士業の業務実態のバランス調整が必要である。長すぎると漏えいリスクが増し、短すぎると再提出・差戻し対応に支障が出る。 | 法務弁護士 + 事業責任者 | Phase 3 着手前（`REQ-504`, `REQ-703`, `RISK-005` の運用設計前。日付は要確認） | `REQ-503`, `REQ-504`, `RISK-005`, バックアップ・DR設計, `docs/roadmap.md` Phase 3〜4（要確認: 指定仕様の `REQ-027` との対応付け） |
| <a id="oq-4"></a>OQ-4 | OCR精度目標値の最終決定 | 要確認。PoC 段階のベンチマーク結果を踏まえて最終的な合格基準を決定する必要がある。現時点の 90% は PoC 着手前の仮値であり、帳票差分や補正工数を踏まえた現実的なしきい値に再設定する余地がある。 | プロダクトオーナー + 技術リード | Phase 1 Go/No-Go 判定前（`REQ-004`, `REQ-701`, `RISK-001`, `RISK-008` の評価確定前。日付は要確認） | `REQ-001`〜`REQ-004`, `RISK-001`, `RISK-008`, Phase 1 到達基準, `docs/roadmap.md` 共通品質評価指標（要確認: 指定仕様の `REQ-005` は現行要件書に未定義） |
| <a id="oq-5"></a>OQ-5 | 法務局提出用様式の具体的受理基準 | 要確認。法務局によって相続関係説明図の受理基準が異なる可能性があり、フォント、余白、記載事項の形式的要件が法令に明示されていない部分がある。地域差・登記官運用差をどこまで吸収するか未決である。 | 司法書士パートナー + プロダクトオーナー | Phase 4 の法務局提出テスト開始前（`REQ-301`, `REQ-704`, `RISK-009` の最終確認前。日付は要確認） | `REQ-301`〜`REQ-304`, Phase 4 の法務局提出テスト, `docs/roadmap.md` Phase 4, `docs/requirements-definition.md` 補足（要確認: 指定仕様の `REQ-016`〜`REQ-019` との対応付け） |
