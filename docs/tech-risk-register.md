# yomitoriAPP 技術リスク登録票

## 運用メモ

- RISK-004 と RISK-010 は近接するが、前者は「外部API送信そのものの情報管理リスク」、後者は「国外移転に伴う法令適合リスク」として分けて管理する。
- RISK-007 と RISK-009 は、コード不具合と法改正未反映の双方が同じ誤判定事象を生みうるため、障害票では原因分類を分離して記録する。
- `要確認` を含む項目は、実装着手前に法務・個人情報保護の確認結果を別紙または台帳で確定させることを前提とする。
